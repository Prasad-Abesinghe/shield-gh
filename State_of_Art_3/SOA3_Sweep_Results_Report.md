# SOA3 — Attacker-Percentage Sensitivity Analysis (Random-Forest IDS)

**Baseline:** Arízaga-Silva, Medina Santiago, Espinosa-Tlaxcaltecatl,
Muñiz-Montero (2025), *"Machine Learning-Powered IDS for Gray Hole Attack
Detection in VANETs"*, World Electric Vehicle Journal (MDPI) 16, 526.
DOI 10.3390/wevj16090526.

**Implementation:** faithful re-implementation of the paper's Random-Forest IDS
(15 estimators, max_depth 15, stratified cross-validation), driven by **real,
event-driven features** from our NS-3 SHIELD-GH simulation. See `README.md`.

---

## 1. What the model actually consumes (real data, not fake)

Every simulation window, `routing.cc` (`soa3_monitor_window()`, enabled with
`--use_soa3_detection=1`) records, per node, the simulation's OWN counters:

| Raw feature | Source in the sim |
|---|---|
| `pkt_received` | `node_total_received[n]` (incremented on real receive events) |
| `pkt_forwarded` | `node_total_forwarded[n]` (incremented on real forward events) |
| `pkt_drop_dp` | `dp_drop_counter[n]` (data-plane gray-hole drops) |
| `pkt_drop_cp` | `cp_drop_counter[n]` (control-plane drops) |
| `is_attacker` | ground truth from the injected attacker sets |

From these, lightweight derived features (`local_pdr`, `drop_rate_dp/cp`,
`fwd_ratio`, network-average PDR and per-node deviation, total drop ratio,
forward efficiency) are computed — mirroring the paper's time/packet/protocol
feature categories. The Random Forest is trained and tested on these real rows.

---

## 2. Purpose of this experiment

We vary the experiment's independent variable — the **attacker percentage** —
and plot how the RF-IDS detection metrics respond. Because `routing_test` is
capped at 5 vehicles, we sweep the four **distinct** attacker densities that the
5-node net supports:

| Attacker % | Real attackers (of 5) |
|---|---|
| 20 % | 1 |
| 40 % | 2 |
| 60 % | 3 |
| 80 % | 4 |

Each simulation is deterministic for a fixed configuration, so the statistical
spread reported by the **95 % confidence intervals** comes from **repeated
stratified k-fold cross-validation** of the real Random Forest over the real
per-window samples (5 splits × 6 repeats = 30 folds) — the paper's own
evaluation protocol (Section 5, 10-fold stratified CV).

| Metric | Definition | Paper |
|---|---|---|
| Detection accuracy | (TP + TN) / N | §5 |
| Precision / Recall (TPR) | TP/(TP+FP), TP/(TP+FN) | Eq. 1–2 |
| F1-score | 2·p·r/(p+r) | Eq. 3 |
| FPR | FP/(FP+TN) | §5 |
| MCC | Matthews correlation coefficient | Fig. 11 / §5 |

Raw data: `results/soa3_real_sweep_results.csv`. Generated by
`soa3_rf_sweep_real.py`.

## 5. Discussion — the metrics genuinely move

- At **low attacker density (20–40 %, 1–2 attackers)** the RF reproduces the
  paper's strong regime: accuracy ≈ 99 %, F1 up to 0.994, MCC ≈ 0.99, zero false
  positives — comparable to the paper's 0.9927 F1 on its balanced dataset.
- As attacker density **rises to 60–80 %**, the malicious class becomes the
  majority and the benign class shrinks. Accuracy and MCC fall sharply (MCC → 0
  at 80 %), and at 80 % the classifier collapses toward predicting the majority
  (attacker) class, so FPR → 100 %. This is precisely the **class-imbalance /
  majority-vote degradation** the paper analyses mathematically (Section 5,
  Eqs. 4–15) — here observed empirically on real simulation data.

This is the point of the exercise the supervisor asked for: with real,
event-driven data fed to the real model, the metrics respond to the independent
variable rather than sitting flat, and the 95 % CIs quantify the confidence in
each point.

---

## 6. Limitations

- The `routing_test` topology is capped at 5 vehicles (Gurobi link-lifetime
  model ceiling), so the sweep resolution in attacker density is coarse
  (1–4 attackers). The trend is nonetheless clear and monotonic in the imbalance
  direction.
- The simulation is deterministic per configuration; run-to-run CIs are not
  available, so the CI is taken over CV folds (the paper's protocol) rather than
  over independent seeds. This is stated explicitly and is the honest CI source
  for a deterministic scenario.
