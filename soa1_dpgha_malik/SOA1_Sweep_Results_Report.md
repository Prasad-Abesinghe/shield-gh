# SOA1 — Attacker-Percentage Sensitivity Analysis (Malik DPGHA)

**Baseline:** Malik, Khan, Qaisar, Faisal, Mehmood (2023), *"An Efficient
Approach for the Detection and Prevention of Gray-Hole Attacks in VANETs"*
(DPGHA), IEEE Access vol.11, pp.46691–46706. DOI 10.1109/ACCESS.2023.3274650.

**Implementation:** faithful re-implementation of the paper's RSU-based
detection (Eq. 13–18), verified against the paper's Table 2 worked example
(see `README.md` / `run_tests.sh`).

---

## 1. Correctness note (what changed)

The earlier SOA1 flagged a node when its PDR fell below `network_average − α`.
**That was not the paper's method.** DPGHA detects two gray-hole variants from
three signals: **PLR** (loss, fixed δ = 3 %), **RRR** (RREP/RREQ ratio, fixed
λ = 70 %), and **μ(DSN)** vs a **dynamic threshold β** (Eq. 16–17). The decision
rule is Eq. 18. The corrected detector reproduces the paper's Table 2 exactly
(β = 49.5; V3 → Smart GHA, V5 → Sequence-Number GHA, all others Normal).

Because this NS-3 setup is data-plane only, PLR is computed from the **real**
forwarding counters while RRR and μ(DSN) are **modelled per node-type** from the
paper's stated gray-hole properties; the detection logic itself is unchanged.

---

## 2. Purpose of this experiment

As with SOA2, we vary the experiment's independent variable — the **attacker
percentage** — and plot how the DPGHA metrics respond. N = 30 vehicles, 30 random
seeds per attacker level (mean ± std), malicious nodes split evenly between Smart
GHA and Sequence-Number GHA. Generated by `dpgha_sweep.py`; raw data in
`results/soa1_sweep_results.csv`.

As in SOA2, honest nodes' observed loss rises with attacker density (downstream
path contamination), so on this small test network the metrics genuinely move
rather than sitting flat.

| Metric | Definition | Paper Eq. |
|--------|------------|-----------|
| Detection accuracy | (TP + TN) / N | §V.F |
| TPR (detection rate) / FPR | TP/(TP+FN), FP/(FP+TN) | Eq. 24 |
| Network PDR | ΣDP_received / ΣDP_sent | Eq. 20 |
| Routing overhead | ΣCP / ΣDP | Eq. 19 |

## 4. Detection accuracy vs attacker percentage

![Detection accuracy](../../results/soa1_sweep_acc.png)

*Figure 2 — DPGHA detection accuracy as the attacker fraction increases.*

Accuracy **rises from ≈ 0.85 at 5 % attackers to 1.00 by 30 %**. At low attacker
density a few honest nodes near the loss threshold are briefly mis-flagged
(FPR ≈ 16 %); as density grows, the dynamic β stabilises and gray-holes separate
cleanly, driving FPR → 0 and accuracy → 1.0. This mirrors the paper's reported
detection-rate behaviour (§V.F).

| Attacker % | Accuracy | FPR |
|-----------:|---------:|----:|
| 5  | 0.848 | 0.163 |
| 15 | 0.966 | 0.040 |
| 30 | 1.000 | 0.000 |
| 60 | 1.000 | 0.000 |

---

## 5. Network packet delivery ratio vs attacker percentage

![Network PDR](../../results/soa1_sweep_pdr.png)

*Figure 3 — Network PDR (Eq. 20) as the attacker fraction increases.*

**The headline trend.** PDR falls **monotonically from ≈ 0.96 at 5 % attackers
to ≈ 0.75 at 60 %**. Detected gray-holes are blacklisted and prevented from
relaying (paper §IV.C), but as attacker density rises more legitimate paths are
contaminated before detection completes, so delivered traffic declines.

| Attacker % | Network PDR |
|-----------:|------------:|
| 5  | 0.96 |
| 20 | 0.91 |
| 40 | 0.83 |
| 60 | 0.75 |

---

## 6. TPR / FPR vs attacker percentage

*(Bottom-left panel of Figure 1.)*

- **TPR (detection rate, Eq. 24) ≈ 1.0** throughout: both gray-hole variants drop
  enough data packets that the PLR > δ gate (combined with RRR ≥ λ or μ(DSN) ≥ β)
  fires reliably.
- **FPR falls** from ≈ 16 % at 5 % attackers to 0 % by 30 %, as the dynamic β
  baseline becomes more representative with more nodes.

---

## 7. Routing overhead vs attacker percentage

![Routing overhead](../../results/soa1_sweep_ro.png)

*Figure 4 — Routing overhead (Eq. 19) vs attacker percentage.*

Routing overhead **rises with attacker percentage** (≈ 0.09 → 0.18). This is the
paper's own mechanism (§V.A): gray-hole nodes generate a large number of RREPs
in response to RREQs (high RRR), inflating control-packet traffic. As the share
of such nodes grows, ΣCP/ΣDP climbs. (Contrast SOA2, where routing overhead is
flat because it charges a fixed per-node control cost.)

---

## 8. Summary

- The metrics respond meaningfully to the independent variable: **PDR degrades**
  and **routing overhead grows** with attacker percentage, while **detection
  accuracy improves** as attacks intensify.
- The detector is now **faithful to Malik et al.** — it reproduces the paper's
  Table 2 worked example exactly, a correctness fix over the previous
  `PDR < avg − α` heuristic.
- **Caveat.** PLR comes from the real simulation counters; RRR and μ(DSN) are
  modelled per node-type because the data-plane sim exposes no control-plane
  signals. The detection logic (Eq. 13–18) is the paper's, unchanged. Wiring real
  RREQ/RREP/DSN counters into the AODV path would remove the modelling and is a
  possible follow-on.
